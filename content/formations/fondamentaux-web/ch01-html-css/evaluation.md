Tu as complété ta modalité (TP Intensif ou Projet Progressif) ? Parfait !

1. **Auto-évaluer** ton travail

3. **Obtenir du feedback**

## Étape 2 : Soumettre

### Format de soumission

Prépare les éléments suivants :

**1. Fichiers source**
```
📁 Dossier de projet (tp1-portfolio/ ou blog-personnel/)
  ├── index.html
  ├── *.html (autres pages)
  ├── assets/css/ (fichiers CSS)
  ├── assets/images/ (images)
  └── README.md (documentation courte)
```

**2. Lien de démo en ligne** (optionnel mais recommandé)
- GitHub Pages : https://username.github.io/repo-name/
- Netlify, Vercel, etc.

**3. Checklist complétée**
Copy-paste de ta checklist d'auto-évaluation avec les scores

### Canaux de soumission

Choisis l'une de ces options :

- 📧 **Email** : dimitri@creative-developer.com
- 💻 **Pull Request** : Sur le repo du projet fil rouge
- 📤 **Formulaire de soumission** : (à définir selon ton préférence)

## Étape 3 : Critères d'acceptation

Ton travail est accepté si :

✅ **Tous les fichiers sont présents** et structurés correctement  
✅ **HTML valide** (W3C Validator, 0 erreur)  
✅ **CSS appliqué** et bien formaté  
✅ **Responsiveness testée** sur DevTools  
✅ **Checklist complétée** avec justifications si critère échoué  
✅ **Score ≥ 7/10**

## Questions courantesd

### Q: Puis-je soumettre incomplètement ?

**R**: Non, ta soumission doit être complète. Si tu as besoin de plus de temps, c'est normal — soumets quand tu es prêt(e).

### Q: Puis-je faire les deux modalités ?

**R**: Oui, c'est possible, mais non obligatoire. Une seule suffit pour progresser. Les deux valident les mêmes compétences différemment.

### Q: Comment sont notés les critères "subjectifs" ?

**R**: Les critères comme "lisibilité" ou "cohérence visuelle" sont évalués selon des standards objectifs (contraste WCAG, espacements cohérents, etc.). Si tu as des questions, demande des clarifications.

### Q: Est-ce qu'on me donne des pistes si j'échoue ?

**R**: Oui, toujours. Le feedback inclut :
- Quel critère échoue
- Pourquoi (avec explications)
- Comment améliorer (ressources, exemples)
